Leo
We should be precise about what this means exactly. The above is an expression that means the output when we ask Replica Mist the question "What do you think about Ike?" Now, perhaps you might be thinking, that the output of Mist when we ask her that question is "Ike? I like him"! And you'd be right. We can notate that with an equal sign:
("What do you think about Ike?") = "Ike? I like him."
Leo
You can read this "the output of Mist when we ask her, 'What do you think about Ike?' is equal to 'Ike? I like him.'" There's also notation for denoting the domain and codomain of a function—oh, but we have to show you how to notate a set. Well, that's easy—just put the elements of the set between curly brackets (remember a set is just a collection of "things"; these "things in a set" are properly called "elements"):
{"What do you think about Ike?", "What do you think about Soren?", "What do you think about Boyd?"}
Leo
But this is extremely long to write out, even with just three elements. So we can also give it a name. Like function names, set names are usually a single letter; we'll use D for "Domain." This is generally done with the word "let" but this a loose rule. The following two equivalent mathematical statements are the main two variations of this that you'll see people use (either the word "be" or an equal sign):
Let D be the set {"What do you think about Ike?", "What do you think about Soren?", "What do you think about Boyd?"}
Let D = {"What do you think about Ike?", "What do you think about Soren?", "What do you think about Boyd?"}
Leo
And then to denote the domain of a function, you use a colon and an arrow. Note also that we're defining our codomain C with very much non-rigorous terminology here ("all possible sentences"). We could make it more formal, but this would be distracting from the point of this example—and indeed often in real mathematics, you'll take shortcuts in mathematical notation in exactly these cases where the full notation would be distracting from your main point and the non-rigorous notation is clear enough anyway.
Let D be the set {"What do you think about Ike?", "What do you think about Soren?", "What do you think about Boyd?"}
Let C be the set of all possible sentences.
: D → C

Numbers

Leo
Perhaps you noticed something strange about the domain and range so far. The range we've been using is an infinite set, one with every possible string of words in it, of any length. The domain, however, has been a finite set of questions. But there's nothing that stops us from letting the domain be infinite, too. In fact, much of mathematics is about functions with infinite domains: that is, functions whose domains are numbers.
Now, you may be concerned about notation. In principle, we could start listing out every possible. In practice, of course, not so much. (Also, even an infinite list is too small to contain all real numbers, but let's not get into that.) So most of the time, we only deal with functions where we can write some sort of rule where we take in a number, do something to that number, and output another number.
